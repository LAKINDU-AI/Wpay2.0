The general flow of Paygold payment is the the following:

  1. The trade user requests the payment of a certain amount through the virtual TPV. In this request the telephone number of the holder or address of the e-mail as the case is case.
  2. The virtual TPV sends an SMS (or e-mail) with a link to the card holder.
  3. The headline clicks on the link and connects to the virtual TPV to enter the payment details (legged, expiration and cvv2). The validity of an operation is not limited, but it is possible to configure the validity time that the client has to make the payment.
  4. The virtual TPV contacts the financial institution that has That card was issued.
  5. If it has established, the issuing entity requests authentication to the card holder.
  6. The virtual TPV requests authorization from the entity broadcast on traditional payment media circuits.
  7. Optionally, the virtual TPV sends a notification On-Line to the trade server with the result of the - I'm running.
  8. Finally, the virtual TPV SIS shows the receipt of the operation to the card holder.

PayGold Parameters

To perform this type of operation via REST, it is necessary that the trade make a request to the TPV-Virtual. After making this request, the TPV-Virtual shall perform the necessary validations and in function of these validations will send a message of reply to the - Trade.

To use this functionality, it is necessary to indicate some fields that are shown below:

  • Ds-Merchant-TransactionType: This field indicates that the type of operation is Paygold, the value of the parameter is "F".
  • Ds.Merchant-Customer-Mobile: To report the the header's phone number to send the SMS to the link.
  • Ds.Merchant-Customer-Mail: To report the email address of the holder to send the link.
  • Ds-Merchant-P2F-ExpiryDate: Allows to specify the expiry date of the link being possible to specify the date of each operation using this parameter. The value of the field may be the number of minutes from the operation or the date and exact expiry date with the following format: aaaa-mm-dd-H.H.MMss.sss (e.g. 2014-08-06-16.31.35.318).
  • Ds.Merchant-Customer-Sms-Text: Customization of SMS messages sent to the user. This field will contain a text chain with the relevant fields between arrobas, is obligatory to always include the URL (URL). The data between arrobas shall be replaced by the corresponding values.
  • Ds.Merchant-P2F-XMLData:Customization of emails sent. This field allows TAGs to pass for the personalization. It is important to escape the symbols and / for their corresponding html code as well as leave them inside the CDATA. The fields that include this parameter are:
    • name Purchaser: The name of the buyer. No character limitation.
    • AddressBuyer: The buyer's address. No character limitation.
    • textLibre1: The text to be included describing the buy. No character limitation.
    • SubjectMailClient: Subject of the personalized e-mail.
